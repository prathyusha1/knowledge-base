  • Scripts: Programs that automate the execution of tasks
  • Web application : client-server software application
  • Web Framework : software framework that is designed to support the development of web applications
  • Application server :
    • Software framework that provides both facilities to create web applications and a server environment to run them
    • hosts and exposes business logic and processes
    • Acts as an intermediate between the web server and the database server
  • Web server :
    • computer system that processes request via HTTP, the basic protocol used to distribute information on the world wide web
    • Deliver web pages and other content to the users
    • serves content to the web using http protocol.
    • Ex - apache
  • Solid Principle :
    • S - Single Responsibility
          §   Class should have only single responsibility
    • O - open closed
          §   Software entities should be open for extension and closed for modification
    • L - Liskov substitution
          §   Objects in a program should be replacable with the instances of their subtypes without affecting the correctness
    • I - Interface aggregation
          §  No client should be forced to depend on methods it does not use. Modularity should be achieved by dividing into small functions
    • D - Dependency Inversion
          §   Decoupling software modules
          §   Abstraction layer should be introduced
  • Server side scripting : employing scripts on a web server which produce response customized for each user request
  • Client Side scripting : changing the interface behavior in response to mouse or keyboard actions or at specified timing events. Content is generated on the user's local system
  • Web Service : web service is a method of communication that allows two software systems to exchange this data over the internet
  • Web API: A Web API is a development in web services
  • Web Resources : documents or files identified by their urls
  • Micro Framework : minimalistic web application frameworks ○ Lacks in functionality of - accounts authentication,authorization,roles..etc ○ Specially designed for building the API's for another service or application
  • Dependency Injection : Technique whereby one object supplies the dependency of other object
  • Stateless protocol: ○ Each request is treated as independent transaction ○ Session information or status of request is not retained ○ Ex : IP, HTTP
  • RESTful API : web services API that adhere to the REST architectural patterns are called RESTFUL api
